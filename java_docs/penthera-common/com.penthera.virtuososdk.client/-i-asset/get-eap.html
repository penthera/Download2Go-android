getEap

abstract fun getEap(): Long

(Expiry After Play or 'EAP') Amount of time this asset will remain available to the user after it has been first played. As soon as possible after EAP has elapsed, Virtuoso will delete this asset. Note: For EAP windowing to work, Virtuoso must know when the asset is first played. It's your responsibility to register a play-start Event with Virtuoso.

Return

Amount of time (in seconds) the asset is to be available after first playout.
