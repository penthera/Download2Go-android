considerAssetHistory

open fun considerAssetHistory(assetHistory: Boolean): PlaylistConfigBuilder

Controls smart-download next download selection logic. Enabling this option will cause smart-download selction logic to consider asset history in addition to Playlist history, when determine what the next item to be downloaded on a Playlist. Example: 1) User downloads, plays, then deletes asset X 2) Next month, A playlist is created for which asset X is a member. VirtuosoPlaylistConfig is created with following settings: isAssetHistoryConsidered = TRUE isPlaybackRequired = TRUE 3) During smart-download find next, asset X will be skipped in Playlist, history shows asset X was downloaded, played, then deleted. Default is FALSE. TRUE: Check asset history FALSE: Do not check asset history.

Return

The playlist builder for chaining.

Parameters

assetHistory

the selected value for consider asset history.
