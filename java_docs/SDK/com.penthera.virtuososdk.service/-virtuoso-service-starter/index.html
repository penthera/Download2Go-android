VirtuosoServiceStarter

abstract class VirtuosoServiceStarter : VirtuosoForegroundServiceHandler

This class starts the service. An application must derive from this and register the class as a broadcast receiver in order to provide the notifications for the service to operate as a Foreground Service. Details of the abstract methods that must be overridden can be found in the abstract base class VirtuosoForegroundServiceHandler. An example of an implementation for this can be found in the demo projects available with Download2Go. The android manifest file will need to contain a definition for the derived service starter class such as: 


     <receiver android:name="com.penthera.sdkdemo.SDKDemoServiceStarter"
			android:enabled="true"
			android:label="SDKDemoServiceStarter"
			android:directBootAware="true">
			<intent-filter>
				<action android:name="android.intent.action.BOOT_COMPLETED"/>
				<action android:name="android.intent.action.QUICKBOOT_POWERON"/>
				<action android:name="com.htc.intent.action.QUICKBOOT_POWERON"/>
				<action android:name="android.intent.action.PACKAGE_REMOVED" />
				<data android:scheme="package"/>
			</intent-filter>
			<intent-filter>
				<action android:name="android.intent.action.BOOT_COMPLETED"/>
				<category android:name="android.intent.category.DEFAULT" />
			</intent-filter>

		</receiver>
