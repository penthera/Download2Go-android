getForegroundServiceNotification

abstract fun getForegroundServiceNotification(context: Context, @Nullable forIntent: Intent): Notification
Content copied to clipboard

This method is required to return a valid notification each time it is called. The notification will be used as the foreground notification for the service, if the method returns null then the download service may not be able to go into foreground mode and continue downloading when the app is in the background. This method works in combination with IForegroundNotificationProvider. If a valid class is returned from getAssetForegroundServiceNotification then that method is used to request all notification updates related to asset downloads and the download engine state. If not then this method will be called for each notification. In either case, this method is called to get a generic foreground notification when the foreground service is first started. That notification will be used where the service goes into foreground mode for reasons other than downloading content. For instance, the service uses foreground mode to protect itself from being shut down prematurely when it is servicing push notification actions or deleting content in the background after expiry. There are some limitations on what can be shown within a notification returned via this method because it is passed across process boundaries. The Notification object implements Parcelable, and in this case it will be added as a Parcellable extra to an Intent. There is a size limit on the total size of objects passing between processes in Android, and that will be applied to this notification.

Return

The notification, which will be passed over a process boundary to be used by the download service.

Parameters

context

A context which can be used in generating the Notification

forIntent

An intent containing the details of the notification required, can be null if the service is being started / restarted and no download context is known.
