The VirtuosoForegroundServiceHandler is a Broadcast Receiver based class which provides the base framework for starting the Download2Go service from the client application. The service is designed to run in a separate process, and to run as a foreground service while downloading so that it can continue to download assets while the app is in the background.

In order to run as a foreground service, a Notification must be shown. The initial notification needs to be generated from the client application but it needs to be set in the service when it enters the foreground state. In order to achieve this, a minimum of the initial notification must be passed across a process boundary from the client application to the service. Further updates to the notification are required to improve the user experience by showing download actions and progress, but these can be generated by calling client application code from within the service. The first notification provided via the service starter is cached within the service as a backup for use if the client code does not provide a valid notification, so it carries special significance.

This abstract class provides the methods which can be used to provide a notification to the service. There is one key method which must be overridden to provide the app notifications and to handle broadcast intents from the service requesting updates to the notification. An initial notification is always delivered via this route in order that the SDK can make the notification available prior to starting the service. The reason for this approach is that the service must start foreground mode, with a valid notification, within a timeout period, otherwise the Android OS will kill the service process.

Following initial startup of the service, the SDK uses com.penthera.virtuososdk.client.IForegroundNotificationProvider to register a class which will be instantiated within the service process and generate notifications directly.
