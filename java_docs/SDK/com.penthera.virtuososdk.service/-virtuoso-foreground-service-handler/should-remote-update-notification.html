shouldRemoteUpdateNotification

open fun shouldRemoteUpdateNotification(aContext: Context, aIntent: Intent): Boolean
Content copied to clipboard

This method determines whether notification update requests received within the service starter should be passed through to getForegroundServiceNotification() and result in a new notification being passed back to the service. This approach to notification updates is deprecated and the method will not commonly be used. This method may be overridden in order to filter on specific intent actions in cases where a notification needs to be generated from the main service starter process and not from within the service. A complimentary method within IForegroundNotificationProvider enables filtering within the service to prevent notifications from being generated on specific intent actions.

Return

True if the notification should be generated and passed to the service, False if the request should be ignored.
