This method has been deprecated under the architectural changes for Android 12. The class implementing com.penthera.virtuososdk.client.IForegroundNotificationProvider should now be specified using a meta-data entry in the manifest with the name "com.penthera.virtuososdk.notification.provider.impl". The class can be instantiated both:

  • In the client process prior to starting a foreground service when the application is in the foreground, and
  • Within the service if it is started from within WorkManager to complete background processes.
This method enables the app to provide a class implementing com.penthera.virtuososdk.client.IForegroundNotificationProvider which will be instantiated within the download service and used for generating Notifications during asset download. This method simply returns the class definition, which will be passed to the service. If a non-null response is provided then the class will be instantiated within the service process and requests for notifications will be sent to it instead of being directed to the getForegroundServiceNotification() method. Notifications generated via that class will not be passed over a process boundary and do not have any size limits imposed by the Parcellable property. The provided class must use a default constructor. Care and consideration should be given to the fact that the class instance will exist in another process to the UI and will need to create duplicate resources within that process for any helper classes used to generate the notifications, such as image loaders or database references. If a class is provided which does not implement the interface then this will be logged and the provided class will be ignored.

Return

The class which should be instantiated within the service to use for notifications.

Deprecated

This has been replaced with a manifest entry to specify the class implemented IForegroundNotificationProvider.
