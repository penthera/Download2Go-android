setProgressUpdateByTime

abstract fun setProgressUpdateByTime(aMilliseconds: Long): ISettings

Configure the frequency of progress updates during a download.

While downloading, Virtuoso will report back updates through the Content Uri, the IQueueObserver or via Broadcasts. This method allows a caller to configure the frequency of those updates by time elapsed during download. The setting represents the minimum amount of time (in ms) that Virtuoso should wait before performing a notification. The lowest permitted value is 1000. Setting a value of 999 or less will reset the update frequency to the default value of 5000. Setting a value of Long.MAX_VALUE will turn off updates on a timed basis. The new setting is applied automatically after being set. Applications can listen for changes to these configuration values by implementing the settingChanged method in IEngineObserver. Flags reported through that API are: SETTING_PROGRESS_UPDATE_TIME

Return

The current settings object, to allow for call chaining.

Parameters

aMilliseconds

The new value to use for configuring the update frequency.
