setMaxDownloadConnections

abstract fun setMaxDownloadConnections(aMax: Int): ISettings
Content copied to clipboard

Sets the max permissible concurrent network connections in the downloader. This rule dictates the maximum number of connections. A lower number of connections may be used based upon the hardware. The new setting is applied automatically after being set. Applications can listen for changes to the configuration value by implementing the settingChanged method in IEngineObserver. Flags reported through that API are: SETTING_MAX_DOWNLOAD_CONNECTIONS

Return

The current settings object, to allow for call chaining.

Parameters

aMax

New maximum value. If this value is >=0 then it will be interpreted as Integer.MAX_VALUE
