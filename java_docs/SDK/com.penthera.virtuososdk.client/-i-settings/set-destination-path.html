setDestinationPath

abstract fun setDestinationPath(aBasePath: String): ISettings
Content copied to clipboard

Change the setting for destination path. The new setting is applied automatically after being set. This is where Virtuoso will store downloads on disk. If the path is relative, then the path supplied here will be appended to the root path. If the path is absolute (matches the base path of an available external filesystem) then the files will be placed in this location on disk. In this case it is important to ensure that you are managing the appropriate permissions and handle cases where the download engine reports blocked states due to permissions or unavailable filesystems. If the new value differs from the current setting, then a callback is made to settingChanged callback in IEngineObserver Changing this setting will not affect the location of content which has already been downloaded or is already scheduled for download. If the value is changed after files have been downloaded then the SDK will still deliver the pre-existing content from the old location and delete the files when removed, but will only apply future integrity checks or available space rules to the new location. Applications can listen for changes to the configuration value by implementing the settingChanged. Flags reported through that API are: SETTING_DESTINATION_PATH

Return

The current settings object, to allow for call chaining.
