ISettings

interface ISettings
Content copied to clipboard

Provides access to the settings which can be configured by the client. The Settings can be accessed through the getSettings call in Virtuoso. Modified settings are automatically saved and there is no need to call the save() method Modifications are buffered when multiple settings values are changed in close succession This allows you to chain modifications together and have the changes persisted in one operation, e.g: 


    ISettings settings = Virtuoso.getSettings().setHeadroom(1024).setMaxStorageAllowed(10240);
Content copied to clipboard
Applications can listen for changes to these configuration values by implementing the settingChanged method in IEngineObserver. Flags reported through that API are detailed in: SettingFlag Get methods in this class return the current setting. If this has just been changed then the new value may not have propagated throughout the SDK, resulting in the old value being returned until the setting is committed and the settingChanged call has been received.
