abstract fun createHLSSegmentedAsset(aManifestUrl: URL, aDesiredBitRate: Long, aAssetId: String, aMetadata: String, aDownloadEncryptionkeys: Boolean, aAddToQueue: Boolean, @Nullable aPermObserver: IQueue.IQueuedAssetPermissionObserver): ISegmentedAsset

Deprecated

Download an HLS video straight from a manifest file. Virtuoso will download and parse the given manifest, select the appropriate profile, and download all the fragments belonging to that profile. This is a blocking operation. If the aAddToQueue parameter is true and aPermObserver is not null then the Permission Observer callback will be called after addition. This method is blocking and will throw an runtime exception if called from the UI thread

Return

Generated ISegmentedAsset, or null if an error occurred

Parameters

aManifestUrl

Remote URL of the manifest

aDesiredBitRate

Virtuoso uses the closest bitrate, which does not exceed this one, when generating from a manifest. A value of 1 means "use the lowest bitrate." If there's no profile of lower bitrate than aDesiredBitRate, Virtuoso will select the lowest bitrate profile. A value of Integer.MAX_VALUE means "use the highest bitrate."

aAssetId

Caller-provided ID that Virtuoso should assign to the asset, will be considered as unique for business logic rules. See IAssetManager summary for more details.

aMetadata

Metadata that Virtuoso should associate with the asset. Typically a JSON string. Can be null.

aDownloadEncryptionkeys

If true, Virtuoso will download all the encryption keys

aAddToQueue

If true, Virtuoso will automatically add the asset to the download queue.

aPermObserver

The observer to be notified of any permission checks when added.
