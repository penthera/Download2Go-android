IAssetManager

interface IAssetManager : IAssetProvider, IAssetAccess
Content copied to clipboard

Provides access to Cursors over the Asset data. The Asset Manager can be accessed through the getAssetManager call in Virtuoso. As well as providing access to commonly grouped Assets: Queued, Downloaded, Expired and Deferred (those which have not yet been added to the queue), the AssetManager provides the API's to create, manipulate and retrieve single Assets.

Assets managed by the system must have an external string identifier upon creation in order for the backplane business logic to function. Assets will be assigned an internal numeric ID and a UUID within a device but these are device specific and will not be consistent across devices. The external identifier, referenced as Asset id, provides consistency across the system. Any assets with the same external identifier will be considered duplicates by the backplane, identifiers should be unique to ensure this does not happen unless intended. This is commonly populated using an id associated with the an external asset catalog.

The Asset Manager's implementation of IAssetProvider defaults to retrieving all Assets know to Virtuoso. The Assets are sorted by their creation time in ascending order. It is possible to issue an update request using the Asset Manager's Uri. The update is restricted to the same fields as the update method:

  1. AssetColumns.METADATA: metadata
  2. AssetColumns.EAP: Expiry after Play
  3. AssetColumns.EAD: Expiry after Download
  4. AssetColumns.START_WINDOW: Start Window
  5. AssetColumns.END_WINDOW: End Window
  6. AssetColumns.CUSTOM_HEADERS: Custom headers
  7. AssetColumns.FIRST_PLAY_TIME: First play time
Any other field passed in the update request will be ignored.
