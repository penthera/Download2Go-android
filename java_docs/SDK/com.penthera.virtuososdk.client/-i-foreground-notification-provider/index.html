IForegroundNotificationProvider

interface IForegroundNotificationProvider

This interface defines the methods which must be provided for the client application to generate update notifications from within the service process. The interface must be declared within the applications Android manifest as a meta-data entry with the name "com.penthera.virtuososdk.notification.provider.impl". For example: <meta-data android:name="com.penthera.virtuososdk.notification.provider.impl" android:value="com.penthera.test.notification.ServiceForegroundNotificationProvider" /> The implementing class must have a default constructor and will be instantiated by the SDK within the * com.penthera.virtuososdk.service.VirtuosoService. It will then be used from within the service * process to request updates to the app notifications. The implementing class will be used in two ways:

  • If the service is started in foreground mode then the notifications will be set for the foreground service using a fixed notification ID that is specified at FOREGROUND_SERVICE_NOTIFICATION_ID . The application may choose it's own Channel ID for foreground service notifications and set that channel in this class.

    The service will call setExistingNotificationForReuse to provide details of the existing foreground service notification that was created during app start up prior to requesting notifications for foreground mode.

  • If the service is started in background mode by WorkManager while the application is not in the foreground then the WorkManager will request a Notification when running on Android versions earlier than API31. The WorkManager notification will be configured with the same notification ID and Channel ID as in Foreground mode to ensure a only one Notification is shown. This class must correctly use the specified Channel in constructing notifications.
Notifications may be switched between foreground and background mode. When this occurs, the method will be called for setExistingNotificationForReuse and the implementation is responsible for repeating any internal configuration to ensure it returns notifications using the appropriate details.
