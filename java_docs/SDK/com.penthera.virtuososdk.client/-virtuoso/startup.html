startup

open fun startup(aUser: String, aExternalDeviceID: String, aPublicKey: String, aPrivateKey: String, aPushRegistrationObserver: IPushRegistrationObserver)

You must call this method so that Virtuoso can register itself with the Backplane. Once this method is successful, you need never call this method again unless updating keys or switching users.

You'll need to call this if AuthenticationStatus==AuthenticationStatus.NOT_AUTHENTICATED.

Registering with the Backplane allows the device to fetch global configuration values for download behavior and windowing rules. It allows Virtuoso to post events to the Backplane and receive push notifications from the Backplane. It allows the Backplane to police the number of allowable download-enabled devices per user.

Successful registration or failures are reported through the requestComplete callback in IBackplaneObserver with a callback type of BackplaneCallbackType.REGISTER. Retrieve the current authentication status by calling authenticationStatus. When calling this method the following holds true: 1) If between calls the User Id changes then the SDK will reset. It will delete all existing files and reset the user data before attempting to register the new user with the Backplane. 2) If between calls ONLY the keys are changed then the SDK will store the new values and attempt to sync in order to get a new license. 3) If between calls the keys and User Id change then the SDK will perform all actions from condition (1) and will also delete any outstanding events that need to be reported to the Backplane. 4) If AuthenticationStatus==AuthenticationStatus.AUTHENTICATED and there are no changes to the keys or User id then this is a NOOP and no callbacks will be performed.

Parameters

aUser

User ID to associate with this Device on the Backplane. The user ID can be a maximum of 512 characters. Backplane registration will fail if it is larger.

aExternalDeviceID

(Optional; can be null) Externally defined device ID. Virtuoso will maintain this so you can refer to this device with this handle.

aPublicKey

Public key used to encrypt HTTP requests made to the Backplane. Get this from Penthera.

aPrivateKey

Private key used to encrypt requests made to the Backplane. Get this from Penthera.

aPushRegistrationObserver

Observer for monitoring whether the push registration is successful
