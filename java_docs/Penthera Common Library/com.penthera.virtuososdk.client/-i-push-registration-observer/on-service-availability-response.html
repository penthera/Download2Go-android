onServiceAvailabilityResponse

abstract fun onServiceAvailabilityResponse(pushService: Int, errorCode: Int)

Called to indicate whether the Push service is available. You must not assume that FCM or ADM is available on all devices. In the case of FCM we recommend that you call GoogleApiAvailability.isUserResolvableError on the reason. If the error is recoverable, you can show a corrective dialog by calling GoogleApiAvailability.makeGooglePlayServicesAvailable. The errorCode can be one of the following ConnectionResult constants defined for GooglePlayServices:

  • SUCCESS
  • SERVICE_MISSING
  • SERVICE_VERSION_UPDATE_REQUIRED
  • SERVICE_DISABLED
  • SERVICE_INVALID
  • DATE_INVALID
*

In the case of ADM the push service will either be supported on the Amazon device or not and no action is required.

Parameters

errorCode

Either the GooglePlayServices ConnectionResult errorCode that uniquely identifies the problem or one of the constants defined in AdmPushServiceAvailability

pushService

The type of push service.
