interface IPushRegistrationObserver
Content copied to clipboard

Enables clients to determine whether a Push Service (Firebase Cloud Messaging - FCM or Amazon Device Messaging - ADM) is available. The client will not be able to receive subscription updates from push notifications until registration successfully finishes. If a recoverable error occurs, the client must handle it by prompting the user to take corrective action. The actions may involve setting up a Google account, installing Google Play Services, registering with an Amazon user. If a transient error occurs, such as a networking problem, the SDK will try to recover the registration ID during its next sync.

Content copied to clipboard

Called to indicate whether the Push service is available. You must not assume that FCM or ADM is available on all devices. In the case of FCM we recommend that you call GoogleApiAvailability.isUserResolvableError on the reason. If the error is recoverable, you can show a corrective dialog by calling GoogleApiAvailability.makeGooglePlayServicesAvailable. The errorCode can be one of the following ConnectionResult constants defined for GooglePlayServices:

  • SUCCESS
  • SERVICE_MISSING
  • SERVICE_VERSION_UPDATE_REQUIRED
  • SERVICE_DISABLED
  • SERVICE_INVALID
  • DATE_INVALID
